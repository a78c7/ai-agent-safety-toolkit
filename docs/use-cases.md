# Use Cases

## 1. Solo Developer Using Codex

A developer runs Testability Doctor before asking Codex to modify a repository, then runs AgentGate before opening a draft PR.

## 2. Maintainer Reviewing AI-Assisted PRs

A maintainer asks contributors to run AgentGate and include the result in PR descriptions when changes were AI-assisted.

## 3. Team Adopting Cursor or Claude Code

A team starts with Testability Doctor to identify which repos are ready for agent work and which need test setup first.

## 4. Developer Evaluating Bounty Issue

A developer runs BountyLens on a bounty issue, checks the repository with Testability Doctor, and only then decides whether to spend agent time.

## 5. Open-Source Project Adding AI PR Guardrails

An open-source project documents that agent-generated diffs should pass AgentGate before draft PR submission.

## 6. Documentation-Only Contribution

A contributor uses Testability Doctor to determine that code changes are risky but documentation changes are reasonable.
