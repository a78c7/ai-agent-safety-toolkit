# Safety Model

The toolkit follows conservative safety boundaries for AI coding agent workflows.

## Shared Principles

- no cookie/keychain/password manager access
- no auto comments
- no auto bounty claim
- no KYC/payment/withdrawal
- no secret upload
- no security exploit tasks
- no auth/payment/crypto/KYC/database migration automation
- human review required
- draft PR preferred

## Practical Meaning

The tools are meant to reduce obvious workflow risk. They do not certify that code is safe, correct, legal, or profitable.

## Human Responsibility

A human reviewer remains responsible for final scope, tests, security judgment, repository policy compliance, and any platform reward process.
